# 下一轮需要你拍板的问题

## 1. 这个项目的公开品牌名

目前我用了 `Advisor Foundry` 作为工作名。

你后面可以考虑三种方向：

- 平台型：`Advisor Foundry`
- 人物型：`Elon Advisor`
- 方法论型：`Source-Grounded Advisors`

## 2. 第一阶段你更想要哪种 demo

三种都可以做，但优先级会影响仓库增长速度：

- `CLI Demo`：最容易先发布，适合开发者
- `Web Demo`：最容易传播，适合拿星
- `Notebook / Evaluation Demo`：最容易建立可信度

## 3. 你要不要把“人物矩阵”提前写进 README

如果一开始就写出未来人物路线图，项目会更像平台：

- Elon Musk
- Jensen Huang
- Lisa Su
- Andy Grove
- Jeff Bezos

## 4. 资料采集的边界要不要更激进

目前这版是保守而稳的：

- 公共来源登记
- 摘要
- 评测
- prompt

如果你要更激进，下一轮可以做：

- 视频转录工作流
- 观点切片数据库
- 检索增强问答

## 5. 你希望这个顾问更偏哪一类价值

这会直接影响 prompt 和 benchmark：

- 战略顾问
- CEO 顾问
- 产品与制造顾问
- 组织与招聘顾问
